Introduction

Ceph is a software storage platform which uniquely delivers object, block, and file storage in one unified system which is highly reliable, easy to manage and free. Ceph delivers extraordinary scalability–thousands of clients accessing petabytes to Exabytes of data.

Ceph was developed by INKTANK storage and the latest stable release is GIANT which is V0.87 which works on Linux, RHEL, Debian, Ubuntu, SUSE etc. The Ceph file system has three main components: the client, each instance of which exposes a near-POSIX file system interface to a host or process; a cluster of OSDs, which collectively stores all data and metadata; and a metadata server cluster, which manages the namespace (file names and directories) while coordinating security, consistency and coherence.


Ceph Installation Process

Steps include:

Add a release key:

wget -q -O- 'https://ceph.com/git/?p=ceph.git;a=blob_plain;f=keys/release.asc' | sudo apt-key add ­

Add packages to repository file (substitute with latest version of Ceph in the highlighted area):

echo deb http://ceph.com/debian-{ceph­stable­release}/ $(lsb_release sc) main | sudo tee/etc/apt/sources.list.d/ceph.list

Update the repository and install ceph­deploy

sudo apt-get update && sudo apt-get install ceph-deploy

Install NTP on all nodes:

sudo apt-get install ntp

Create a Ceph user on all the nodes (including admin, OSD's, and monitor nodes) and ensure the user has sudo privileges:

ssh user@ceph-server
sudo useradd -d /home/{username} -m {username}
sudo passwd {username}

Enable password-less sudo access for the new user:

echo "{username} ALL = (root) NOPASSWD:ALL" | sudo tee /etc/sudoers.d/{username} sudo chmod 0440 /etc/sudoers.d/{username}
